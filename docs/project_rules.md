# 프로젝트 문서·구현 규칙 (글로벌 룰)

- **문서화**: 논의·구현 내용은 markdown으로 프로젝트 서브디렉토리에 정리한다. 구현은 **자세히** 적어 이후 코드 분석에 쓸 수 있게 한다.
- **문서 참고**: 구현 요청 시 관련 문서를 먼저 확인해 정확히 반영한다. 새 대화에서도 기존 문서를 보고 진행한다.
- **문서 정리**: 필요 없어진 문서는 정리·삭제한다. 문서만 늘리지 않는다.
- **디렉토리 구분**: 사용자 설명용 문서와 **구현 관리용 문서**는 디렉토리를 나눠 둔다.
  - 구현·관리: `docs/impl/` (각 스크립트/모듈별 구현 요약, CLI, 데이터 흐름 등)
  - 설명·튜토리얼: 필요 시 `docs/` 또는 별도 디렉토리
- **예외 처리**: 기본적으로 예외 처리를 추가하지 않는다. 사용자가 명시할 때만 추가한다.
- **import 실패**: 사용자가 명시하지 않는 한 import 실패에 대한 예외 처리는 하지 않는다.
