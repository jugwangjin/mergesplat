# Trisplat: O(1) 동적 아틀라스 업데이트 (QEM Edge Collapse용)

나중에 trisplat에서 simplification/densification( QEM edge collapse + texture atlas ) 구현 시 참고할 설계 문서.

---

## 문제

QEM(Quadric Error Metric) 기반 에지 붕괴(Edge Collapse) 시 텍스처 아틀라스를 갱신하는 것은 전통적으로 **$O(N \log N)$ 이상의 전역 연산**이다. 위상이 바뀔 때마다 xatlas 등으로 전체 메쉬 UV를 다시 펴고 거대한 이미지를 재배치(repacking)하면 **미분 가능 렌더링 루프가 멈춘다**.

---

## 제안: O(1) 동적 아틀라스 업데이트

운영체제의 **메모리 페이징(Memory Paging)** 과 딥러닝의 **텐서 리샘플링**을 결합한 방식.

---

### 1. 사전 준비: 고정 크기 타일 메모리 풀 (Tile Memory Pool)

- 하나의 거대한 48D SH 텍스처 아틀라스가 아니라, **고정 해상도(예: $16 \times 16$)의 독립 타일들의 1차원 배열**로 정의.
- **Free List**: 사용 가능한 타일 인덱스(ID)를 스택(Stack)으로 관리.
- **UV 독립성**: 메쉬 전체가 하나의 거대한 UV 맵을 쓰지 않음. 각 삼각형은 **할당된 타일 번호(Tile ID)** 하나만 가지며, 삼각형의 세 정점 UV는 **해당 타일 내부의 국소 좌표** $(0,0), (1,0), (0,1)$만 가리킴.

---

### 2. 에지 붕괴 및 새 타일 할당 (Allocation: $O(1)$)

- QEM이 정점 $v_1$, $v_2$를 $v_{new}$로 붕괴시키기로 결정.
- 두 정점을 공유하던 삼각형 2개는 **소멸**.
- 주변에 연결된 $k$개 삼각형은 꼭짓점 하나가 $v_{new}$로 바뀌며 **형태 변형(deformation)** → 텍스처 늘어짐(stretch) → 텍스처 갱신 필요.
- **새 타일 할당**: `Free_List.pop()`을 $k$번 호출해 빈 타일 $k$개를 가져옴. **$O(1)$** (스택 연산, $N$과 무관).

---

### 3. 미분 가능한 국소 리샘플링 (Local Baking: 텐서 연산)

- 새로 할당된 $k$개 빈 타일에, **변형 전** 원래 삼각형들이 갖고 있던 48D SH 데이터를 옮겨 담음.
- **PyTorch `F.grid_sample`** 로 GPU에서 병렬 처리.
  1. **픽셀의 3D 좌표화**: 새 타일의 texel 중심을 **무게중심 좌표**로 3D $(x,y,z)$로 역투영(unprojection).
  2. **구형 표면으로의 레이캐스팅**: 그 3D 점들을 **에지 붕괴 직전**의 구형 삼각형들(소멸한 2개 + 변형 전 $k$개) 표면에 가장 가까운 지점으로 매핑.
  3. **색상 복사**: 매핑 지점의 구형 UV를 구하고, 구형 타일에서 48D SH를 **bilinear interpolation**으로 읽어 새 타일에 기록.
- 거리 계산·보간만 쓰므로 **미분 그래프가 유지**되고, QEM 병합 중에도 텍스처가 **역전파 가능**하게 학습됨.

---

### 4. 가비지 컬렉션 (Garbage Collection: $O(1)$)

- 베이킹이 끝나면, **소멸한 삼각형 2개**와 **갱신 전 구형 삼각형 $k$개**가 쓰던 예전 타일들은 불필요.
- 해당 타일 ID들을 `Free_List.push()`로 풀에 반환. **$O(1)** (스택 삽입, 정렬/탐색 없음).

---

## 요약·학술적 강점

- 일반적인 UV 패킹(테트리스식 빈 공간 채우기)은 **NP-Hard**라 렌더 루프 안에서 실시간 수행 불가.
- **타일 풀 + 독립 할당**을 쓰면, 아틀라스 이미지를 옮기거나 리사이즈할 필요 없이 **Tile ID만 스왑**으로 관리 가능.
- **메모리 파편화**를 막으면서 연산량을 낮춰, **3DGS/메쉬 기반 미분 가능 렌더링**에 QEM을 통합하는 핵심 설계로 활용 가능.

---

## Trisplat 연관

- 계획상 **TriangleModelAtlas**(per-triangle 16×16 alpha/SH/opacity) + **QEM edge collapse**(L2 color + opacity 메트릭) 단계에서, 위 타일 풀·O(1) 할당/반환·미분 가능 국소 베이킹 설계를 적용할 예정.
